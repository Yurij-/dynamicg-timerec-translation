Time Recording - Holiday tracking


Track your holidays using one of these two options. Option A1 is easier to handle, option A2 makes sure your "target time" doesn't deviate.

(A1) Using day notes
• Data input (screen 1): put in "Holiday" or similar as day note. Hint: use the "[...]" lookup to select and edit template texts.
• Reporting (screens 2 and 3): open report E1 using day notes filter 'Holiday'. Hint: "[...]" allows reusing previously applied filters.



(A2) Time block with dedicated task
• Data input (screen 1): put in a work unit that matches your regular work time (e.g. 0900 to 1700 without breaks) and assign "Holiday" as task.
• If you haven't set up task "Holiday" yet: long click on the "Task" node opens the task edit page as per screen 2.
• Reporting (screen 3): open report E1 using task filter 'Holiday'. Hint: "[...]" allows reusing previously applied filters.



(B) Public holidays
Use same approach as above but set different notes text or task (e.g. "Public Holiday").
Alternatively, leave that day just empty (but see next section)
• Hint: If tracking paid holidays use option (A2) as stated above and make sure you set the correct amounts on that task (using either hourly rate, or "fixed amount per day").

(C) Reduced weekly working hours vs. "Weekly target time"
If "Weekly target time" is enabled in the app you'll need to adjust the regular work time on those weeks with public holidays - see Preferences on page 3.
• Hint: Long clicking the "weekly delta" cell on the main screen also opens the "Override weekly target time" screen.



(D) Short working days
The daily target time (as per Preferenes page3, "Daily target time") can be changed specifically for single days - use this when working on half-day public holidays or similar.
